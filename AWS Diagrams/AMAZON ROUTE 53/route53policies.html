Routing Policy Summary Table

Routing Policy Use Case How it Works
Simple Routing Directs traffic to a single resource. Single IP address or server.
Weighted Routing Distribute traffic across multiple resources with specific weight. Routes traffic based on assigned weights.
Latency-Based Routing Directs users to the region with the lowest latency. Routes traffic to the lowest latency region.
Geolocation Routing Routes traffic based on the userâ€™s geographic location. Routes traffic to a specific region or country.
Geoproximity Routing Routes traffic based on both geographic location and proximity to resources. Routes traffic with optional bias toward certain locations.
Failover Routing Routes traffic to backup resources when primary resources fail. Routes traffic to failover resources based on health checks.
Multivalue Answer Routing Routes traffic to multiple healthy records. Returns multiple IP addresses for clients to choose from.
